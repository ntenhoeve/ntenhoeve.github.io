Introspect Framework Concepts

The Introspect class needs to be initialized before it can be used. This is done once using the IntrospectInitializer interface. This interface basically provides the implementations of the different provider interfaces.

You can choose to implement this interface your sel f , but t o make things easy, you can also choose to use one of the default implementations of the IntrospectInitializer interface depending on the type of application you are writing:
  • A command line application ( CommandLineIntrospectInitializer )
  • A desktop application ( SwingIntrospectInitializer )
  • A mobile application ( AndroidIntrospectInitializer )
  • A web application ( VaadinIntrospectInitializer )
  • A command line application ( CommandLineUserInterfaceProvider )
  • A desktop application ( SwingUserInterfaceProvider )
  • A mobile application ( AndroidUserInterface )
  • A web application ( VaadinUserInterface )

Steps to initialize the introspect framework/ your application .
  • Choose the IntrospectInitializer implementation you need.
  • service objects .
  • Initialize the introspect framework with the Introspect.initialize(yourIntrospectInitializer)
